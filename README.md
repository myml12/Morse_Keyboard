# Morse Keyboard

XIAO ESP32-S3 を BLE キーボードとして使う、モールス電鍵キーボードです。パドルの短点・長点を Iambic A キーヤーで生成し、モールス符号を英数字・記号へ復号して、スマホやPCへ送信します。

ファームウェアは [firmware.ino](firmware.ino) だけで完結しています。このREADMEは、マイコン側のコードを上から順に解説するための資料です。

## 仕様

| 項目 | 内容 |
| --- | --- |
| 対象ボード | Seeed Studio XIAO ESP32-S3 |
| キーヤー方式 | Iambic A |
| 初期速度 | 22 WPM |
| 初期比率 | Dot:Space:Dash = 1:1:3.0 |
| 出力 | BLE HID キーボード、アクティブブザーのサイドトーン |
| 入力文字 | A–Z、0–9、モールス表に登録した記号 |

Iambic A は、両パドルを押したスクイーズ中には短点・長点を交互に送りますが、両方を離した後に追加の要素を送らない方式です。離した後に余計な短点／長点が付くことを避けたい用途に向いています。

## 配線

パドルは「押すとGNDへ短絡する」接続です。共通端子をGNDへ、各接点を対応するGPIOへ接続します。

| 用途 | XIAOピン | ESP32 GPIO | 接続先 |
| --- | --- | --- | --- |
| Dit（短点） | D1 | GPIO2 | 短点接点 |
| Dah（長点） | D6 | GPIO43 | 長点接点 |
| サイドトーン | D9 | GPIO8 | アクティブブザーの信号入力 |
| パドル共通 | GND | — | パドル共通端子 |

入力は `INPUT_PULLUP` を使うため、未押下は `HIGH`、押下は `LOW` です。

> D6/GPIO43 は UART TX と共用です。このファームは `Serial` を一切使わないことでD6をパドル入力に使っています。デバッグ用に `Serial.begin()` や `Serial.print()` を追加すると、長点入力と競合する可能性があります。

長い配線、金属製パドル、強いノイズ源の近くで使う場合は、D1・D6から3.3 Vへ各 4.7〜10 kΩ の外部プルアップ抵抗を追加してください。

## 導入と書き込み

### 本体（ESP32-S3 / `firmware.ino`）

1. Arduino IDEに XIAO ESP32-S3 用のESP32ボードパッケージを導入します。
2. [ESP32-BLE-Keyboard](https://github.com/T-vK/ESP32-BLE-Keyboard) ライブラリを導入します。
3. `firmware.ino` をArduino IDEで開き、ボードに **XIAO ESP32-S3** を選択して書き込みます。
4. スマホまたはPCのBluetooth設定から **XIAO Morse Keyboard** をペアリングします。

### 別基板（ESP32-C3 / `firmware_2/`）

短点 **D0** / 長点 **D1** / 他励式ブザー **D8**。BLE名は **XIAO Morse_2**。

```text
firmware_2/firmware_2.ino
```

Arduino IDE で `firmware_2` フォルダを開き、ボードに **ESP32C3**（使用ボードに合わせて）を選んで書き込みます。他励式ブザーは LEDC（700 Hz）で駆動します。

BLE未接続中もパドルとブザーは動作しますが、復号した文字は送信されずに破棄されます。

## 全体の処理の流れ

```text
パドル接点
  ↓ digitalRead + PaddleInput
短点／長点の押下・離しイベント
  ↓ IambicKeyer
Dit / Dah の一定長マーク
  ↓ Sidetone
ブザーON/OFF + MorseDecoderへ要素を通知
  ↓ MorseDecoder
3単位の無音で文字確定、7単位の無音でスペース
  ↓ writeKeyboard
BLE HIDキーボードとしてスマホ／PCへ送信
```

`loop()` は約1 msごとにこの流れを進めます。ブロッキングする長い `delay()` は使わず、各要素の終了時刻を `millis()` で管理します。

## 設定定数

`firmware.ino` の先頭にある定数だけを変更すれば、配線や操作感を調整できます。

| 定数 | 初期値 | 説明 |
| --- | ---: | --- |
| `DOT_PIN` | `D1` | 短点接点のGPIO |
| `DASH_PIN` | `D6` | 長点接点のGPIO |
| `BUZZER_PIN` | `D9` | アクティブブザー出力 |
| `SWAP_PADDLES` | `false` | 短点・長点の物理配線が逆なら `true` |
| `WPM` | `22` | 送信速度。変更後は再書き込みが必要 |
| `DASH_RATIO` | `3.0f` | 長点の長さ。Icom系の 1:1:X ではXに相当し、2.8〜4.5を推奨 |
| `CONTACT_DEBOUNCE_MS` | `20` | 押下を有効と判定するまでの安定時間 |
| `RELEASE_REARM_MS` | `20` | 離し際の再チャタリングを次の押下として受け付けない時間 |

### WPMと各時間

`dotMs()` は短点の長さを次式で求めます。

```text
短点時間 [ms] = 1200 / WPM
```

初期値の22 WPMでは、短点と要素間隔は約54 ms、長点は約162 msです。文字は最後の要素から3短点分の無音で確定し、7短点分の無音で単語区切りのスペースを送ります。

## コード解説

### 1. ライブラリとBLEキーボード

```cpp
#include <Arduino.h>
#include <BleKeyboard.h>
#include <cstring>
#include <driver/gpio.h>
```

- `Arduino.h` はGPIO、`millis()`、`delay()` を提供します。
- `BleKeyboard.h` はESP32をBLE HIDキーボードとして動かします。
- `cstring` はモールス文字列の比較に使う `strcmp()` のためのものです。
- `driver/gpio.h` はD6をUART機能から切り離す `gpio_reset_pin()` に使います。

```cpp
BleKeyboard bleKeyboard("XIAO Morse Keyboard", "Y.Mizuno", 100);
```

このオブジェクトがBluetooth機器名、製造者名、バッテリー残量100%を宣言します。スマホ側には **XIAO Morse Keyboard** と表示されます。

### 2. `writeKeyboard()` — BLE文字送信

`writeKeyboard()` は、復号済みの1文字をBLE HIDへ送る唯一の関数です。未接続なら何も送らず、接続中なら `bleKeyboard.write()` で1文字送ります。

### 3. `MORSE_TABLE` — 符号表

`MorseEntry` は、モールス文字列と送信するASCII文字の組です。

```cpp
struct MorseEntry {
  const char* code;
  char character;
};
```

`MORSE_TABLE` には、以下を登録しています。

- 英字: `A`〜`Z`
- 数字: `0`〜`9`
- 記号: `. , ? ' ! / ( ) & : ; = + - _ " $ @`

文字は大文字で送信されます。表に存在しない符号、または7要素を超える符号は文字として送信されません。

### 4. `MorseDecoder` — モールス符号の復号

`MorseDecoder` は、送信済みの短点・長点を `marks_` に蓄積し、無音時間で文字や単語を確定します。

| メンバー | 役割 |
| --- | --- |
| `marks_` | `.` と `-` を最大7要素まで保持する配列 |
| `marksLength_` | 現在の要素数 |
| `characterPending_` | 文字確定待ちかどうか |
| `characterSent_` | 少なくとも1文字を確定済みか |
| `spaceSent_` | 現在の単語区切りスペースを送信済みか |
| `lastElementEndMs_` | 最後の要素が終わった時刻 |

#### `addElement(bool dah, uint32_t now)`

短点または長点が終わるたびに `Sidetone::end()` から呼ばれます。`dah` が `false` なら `.`、`true` なら `-` を追加し、文字確定待ち状態にします。

#### `flush(uint32_t now)`

無音時間を確認します。

- 3短点分以上の無音: `marks_` を `MORSE_TABLE` と照合して1文字を送信し、バッファを空にします。
- 7短点分以上の無音: スペースを1つ送信します。

`flush()` は通常の `loop()` だけでなく、次のマーク開始直前にも呼ばれます。これにより、次の文字を打ち始めた瞬間に前の文字やスペースが失われません。

### 5. `Sidetone` — ブザーと復号器の橋渡し

このプロジェクトは **アクティブブザー** 用です。ブザーは `HIGH` で鳴り、`LOW` で止まる前提です。

- `begin()` は、必要なら直前の文字を確定してからブザーを `HIGH` にします。
- `end()` はブザーを `LOW` にし、終わった要素を `MorseDecoder::addElement()` へ渡します。

パッシブブザーは単に `HIGH` にしても所望の音程では鳴らないため、このままでは使えません。パッシブブザーを使う場合はPWM／LEDCによる発振処理が必要です。

### 6. `IambicKeyer` — 短点・長点を一定長で送る状態機械

`IambicKeyer` は次の3状態を持ちます。

| 状態 | 内容 |
| --- | --- |
| `Idle` | パドル待ち。押下があれば次の要素を開始 |
| `Mark` | 短点または長点を送信中 |
| `Gap` | 要素間の1短点分の無音 |

#### `setPaddle(element, pressed)`

`element` は `0` がDit、`1` がDahです。パドルの確定した押下・離し状態を保持します。Idleからの最初の押下は `first_` に保存し、同時押しを開始したときにも最初に押した側を最初の要素にします。

#### `nextElement()`

要素間隔が終わったとき、次の要素を決めます。

| パドル状態 | 次の動作 |
| --- | --- |
| Ditのみ押下 | 短点を繰り返す |
| Dahのみ押下 | 長点を繰り返す |
| 両方押下 | 前回と反対の要素を送る |
| 両方離した | 次の要素を送らず停止 |

最後の行が Iambic A の要点です。Iambic Bのような「離した後の追加1要素」は実装していません。

#### `tick(now)`

`deadlineMs_` と現在時刻を比較し、Mark終了、Gap終了、次要素開始を進めます。パドルを離しても、すでに始まったMarkは短点または長点の規定時間まで完了します。ただし離した状態は直ちに反映されるため、その次の要素は開始しません。

### 7. `PaddleInput` — 接点チャタリング対策

機械式接点は押下／離し直後に短時間ON/OFFを繰り返すことがあります。`PaddleInput` は各パドルにつき1つずつ存在し、`update()` が入力状態の変化を返します。

| 操作 | 処理 |
| --- | --- |
| 押下 | `CONTACT_DEBOUNCE_MS`（初期値20 ms）連続して押されてから有効化 |
| 離し | 即時に無効化し、キーアーへ伝達 |
| 離し直後の再押下 | `RELEASE_REARM_MS`（初期値20 ms）経過するまで無視 |

離しを待たずに即時反映するのは、5個目の短点を離したあとに6個目が始まる問題を防ぐためです。進行中の1要素は完了させつつ、次の要素だけを止めます。

### 8. `configureInput()` と `updatePaddles()`

#### `configureInput(pin)`

`gpio_reset_pin()` でUARTなどの既存機能を外し、`INPUT_PULLUP` に設定します。特にD6をパドル入力として使うために必要です。

#### `updatePaddles(now)`

1ループでD1とD6を各1回だけ読みます。`SWAP_PADDLES` に従って短点・長点へ割り当て、各 `PaddleInput` の状態変化だけを `IambicKeyer::setPaddle()` へ渡します。

同じGPIOを複数回読む処理を避けているため、素早いスクイーズを片側だけの押下として誤認しにくくしています。

### 9. `setup()` と `loop()`

#### `setup()`

1. D1/D6を入力プルアップへ設定します。
2. D9を出力にし、ブザーを停止します。
3. `bleKeyboard.begin()` でBLE HIDサービスを開始します。

#### `loop()`

```cpp
const uint32_t now = millis();
updatePaddles(now);
keyer.tick(now);
if (!keyer.sending()) decoder.flush(now);
delay(1);
```

この順序には意味があります。先に接点状態を確定してからキーアーを進めるため、離した瞬間が次の要素開始より優先されます。送信中でないときだけ通常の文字／単語間隔を確認し、最後に1 msだけCPUを譲ります。

## 操作例

| 入力 | モールス符号 | 送信文字 |
| --- | --- | --- |
| 短点を1回 | `.` | `E` |
| 長点を1回 | `-` | `T` |
| 短点→長点 | `.-` | `A` |
| 短点を3回 | `...` | `S` |
| 短点を5回 | `.....` | `5` |

文字間は3短点分、単語間は7短点分待ちます。自動キーイング中は、片側を保持するとその要素を繰り返し、両側を保持すると交互に送ります。

## トラブルシュート

| 症状 | 確認・対処 |
| --- | --- |
| 短点と長点が逆 | `SWAP_PADDLES = true` に変更 |
| 離した後に余計な要素が出る | 接点配線とGND共通を確認。外部プルアップを追加し、必要なら両デバウンス値を少し増やす |
| 長点だけ不安定 | D6にSerial出力を追加していないか確認。改善しなければD2など別GPIOへ配線し、`DASH_PIN` を変更 |
| ブザーが鳴らない | アクティブブザーか確認。極性、GND、D9の配線を確認 |
| BLE接続できない | 既存のペアリングを削除してから再ペアリング。電源再投入後に再試行 |
| 日本語になってしまう | スマホ側で英語キーボードを選ぶ |
| 文字が送られない | BLE接続を確認。未接続中に打った符号は保存されない |
| 文字が意図せず確定する | WPMを下げるか、各文字間の無音を3短点分より短くする |

## macOSシミュレータ

`sim/` には、ファームと同じ既定値（Iambic A / 22 WPM / 1:1:3.0）で試せるmacOS用シミュレータがあります。シミュレータ上では他のキーアー方式も選択できます。

```sh
cd sim
make
./morse_simulator
```

| キー | 役割 |
| --- | --- |
| `D` | Dit（短点） |
| `F` | Dah（長点） |
| `Space` | Dit（短点。Dと同じ） |

`make clean` でビルド成果物を削除できます。
